# 产品设计规范 · 设计原则 2.0

> 本文档定义 ANI Console / BOSS 的产品设计原则，作为页面、组件、交互与评审的上位约束。  
> 实现层组件库：**TDesign React 1.x**（唯一指定）。

---

## 1. 设计定位

### 1.1 适用产品

- ANI Console：租户侧算力、模型、推理、知识库、存储等资源管理
- ANI BOSS：平台运营、租户与配额、安装交付、系统监控
- 同类 B 端：高信息密度、强任务闭环、强调状态感知与操作效率

### 1.2 设计强调的能力

- **桌面优先**：支持复杂业务操作与多区域协作（列表 + 详情、表格 + 检查器）
- **层级清晰**：通过 TDesign 面板、表格、表单与语义色组织信息，而非强装饰
- **状态可感知**：让用户快速理解当前对象、资源状态与下一步动作
- **AI 为增强**：知识库问答、诊断建议等作为可选侧栏/面板，不替代主页信息架构

### 1.3 不适合直接照搬的场景

- 品牌官网、营销活动页
- 内容阅读型产品
- 移动端优先的消费级应用（Phase 3 移动端另用 TDesign Mobile，不在本文范围）

## 2. 核心原则

### 2.1 任务优先

- 页面先回答「用户现在要做什么」，再考虑视觉表现
- **主操作**必须一眼可见；次操作明显退后（TDesign `Button theme="primary"` 唯一主按钮）
- 用户应能在 **3 秒内**识别：当前对象、状态、下一步动作

### 2.2 语义一致

- 同一颜色表达同一类语义，跨页面含义不变（对齐 TDesign 状态色 Token）
- 相同区域承担相同职责：顶栏、侧栏、页头、主工作区、辅助区、反馈区
- 列表、详情、弹窗、通知的行为与视觉保持稳定

### 2.3 复用优先（TDesign 优先）

- 新页面 **优先选用 TDesign 标准组件**，再考虑 `src/components/` 业务封装
- 禁止在单页内发明一次性按钮样式、独立间距体系或平行色板
- 确需新样式时：先评估 TDesign `variant` / `theme` / 主题 Token 是否已覆盖；仍不足则沉淀为 **共享业务组件** 并更新规范

### 2.4 可访问性优先

- 图标按钮必须提供可访问名称（`aria-label` 或可见文本）
- 焦点态必须可见（TDesign 默认 focus 样式不得移除）
- 表单必须有 `label` 或等价可访问名称
- 状态与错误反馈 **不能仅靠颜色**（需配合图标、文案或 `Alert` 组件）

## 3. 视觉策略

### 3.1 风格基调

- **克制、稳定、任务导向**；面板、表格、表单强调可读性
- 装饰性动画极少使用；Loading 使用 TDesign `Loading` / `Skeleton`，避免自定义闪烁动画
- Console 壳层参考云控制台：**品牌色顶栏 + 浅色侧栏 + 高密度工作区**（见「页面模板 2.0」壳层章节）

### 3.2 色彩策略

设计阶段使用 **设计语义**，交付与实现必须映射到 **TDesign Token**（详见《TDesign 组件与 Token 2.0》）。

| 设计语义 | 用途 | TDesign 实现参考 |
|----------|------|------------------|
| 页面底色 | 应用背景 | `--td-bg-color-page` |
| 容器/卡片 | 面板、卡片 | `--td-bg-color-container` |
| 主文本 | 标题、正文 | `--td-text-color-primary` |
| 次级文本 | 说明、元信息 | `--td-text-color-secondary` |
| 占位/禁用文本 | placeholder | `--td-text-color-placeholder` |
| 品牌/主操作 | 主按钮、顶栏、链接强调 | `--td-brand-color` 及 `-light` / `-focus` |
| 边框 | 分割、输入框 | `--td-component-border` |
| 成功 | 正常/成功状态 | `--td-success-color` |
| 警告 | 需关注 | `--td-warning-color` |
| 错误/危险 | 错误、破坏性操作 | `--td-error-color` |

**规则：**

- 设计稿标注 Token 名；避免仅标注 `#RRGGBB`（品牌色定制时除外，且须同步主题配置）
- 状态色只表达状态，不做大面积装饰底
- Hover / focus / active 遵循 TDesign 组件内置交互态，不在页面层重复定义

### 3.3 字体与字号

**字体栈（与 TDesign 中文 B 端一致）：**

```text
PingFang SC, Microsoft YaHei, -apple-system, BlinkMacSystemFont, Segoe UI, sans-serif
```

**推荐层级（设计参考；实现优先用 TDesign Typography / 组件默认字号）：**

| 层级 | 建议字号 | 字重 | 典型用途 |
|------|----------|------|----------|
| 页标题 | 20px | 600 | Page Header 主标题 |
| 页副标题 | 14px | 400 | 上下文说明 |
| 正文 | 14px | 400 | 表单、段落 |
| 区块标题 | 16px | 600 | Card / 分组标题 |
| 指标数字 | 24–28px | 600 | Statistic、概览 KPI |
| 表头 | 14px | 600 | Table 列头 |
| 元信息 | 12px | 400 | 时间、ID、辅助说明 |

**规则：**

- 标题承担结构层级，不是单纯放大字号
- 数值对比区优先等宽数字（Statistic、表格数字列）
- 不在业务页混用第三套字号体系

### 3.4 间距、圆角与阴影

**间距节奏（4px 基准，与 TDesign 栅格一致）：**

```text
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40
```

| 场景 | 推荐间距 |
|------|----------|
| 页面主内容区内边距 | 16–24px |
| **页面级区块之间** | **16px**（Alert、指标行、分区网格、页头与主内容；`ConsolePage` 栈间距，与 Row gutter 一致） |
| 同层 Row/Col 栅格 gutter | 16px |
| 卡片内边距 | 16–24px |
| 表单项之间 | 16–24px |
| 行内按钮、Tag 间距 | 8px |
| 密集表格单元格 | 8–12px |

**圆角：**

- 卡片、面板、弹窗：`8px`（对齐 TDesign `--td-radius-medium` 量级）
- 按钮、输入框：遵循 TDesign 组件默认
- 不在单页使用 16px+ 大圆角破坏控制台气质

**阴影：**

- 仅用于 `Dialog`、`Drawer`、下拉浮层
- 普通 `Card`、表格区域优先 **边框 + 背景分层**，少用阴影

## 4. 主题模式

### 4.1 当前优先级

| 优先级 | 要求 |
|--------|------|
| **P0（必须）** | Light 主题完整可用 |
| **P1（规划）** | Dark 主题或深色顶栏 + 浅色内容区组合；Token 名保持一致 |

TDesign 支持主题切换。设计 dark 稿时，所有语义色须在两套主题下 **可区分、可阅读**，不能只换背景不换边框与次级文本。

### 4.2 设计师禁止事项

- 按主题分支设计两套完全不同的组件形态
- 为 dark 单独发明非 TDesign Token 的色板

## 5. 交互原则

### 5.1 操作层级（映射 TDesign Button）

| 设计层级 | TDesign 用法 | 场景 |
|----------|--------------|------|
| 主操作 | `theme="primary"` | 每操作组唯一 |
| 次操作 | `theme="default"` 或 `variant="outline"` | 取消、刷新、导出 |
| 弱操作 | `variant="text"` | 表格行内次要链接 |
| 危险操作 | `theme="danger"` | 删除、卸载、回收 |
| 链接式 | `variant="text"` + 品牌色 | 跳转详情 |

### 5.2 异步反馈

- 进行中：`Button loading`、`Loading`、`Progress`
- 成功：`Message.success` 或页面内 `Alert theme="success"`
- 失败：`Message.error` / `Alert`，含 **原因 + 建议**（重试、联系管理员、检查权限）

### 5.3 行为安全

- 危险操作必须 `Dialog` 二次确认，说明影响范围
- 表格 **行点击** 与 **行内按钮** 区域分离，避免误触
- 行选中、行跳转、快捷操作优先级在模板中写清（见页面模板 2.0）

### 5.4 响应式

- 桌面优先（≥1280px 为设计基准宽度）
- 窄屏时辅助面板 `Drawer` 化或下移
- 表格允许横向滚动，**关键操作列** 优先固定或保持在首屏

## 6. 文案与术语

### 6.1 文案规则

- 按钮用 **明确动词**：「创建知识库」「导入模型」「删除实例」
- 避免单独使用「继续」「提交」「更多」（除非上下文已极明确）
- 空态、错误态文案能 **独立读懂**，不依赖口头解释

### 6.2 术语来源

- Console 模块名、资源名与 `repo/services/docs/console-modules/` 保持一致
- Core 资源（instances、networks、volumes 等）与 OpenAPI 字段.display 名对齐
- 同一资源在中英文下 **含义一致**（i18n 为 P1，但设计时需预留英文更长文案空间）

---

## 7. 可访问性要求（设计侧）

设计稿与原型须满足：

- 核心流程可键盘完成（Tab 顺序合理）
- 焦点顺序与视觉顺序一致
- 表单字段均有关联标签
- 错误提示靠近字段，不仅用红色边框
- 图表关键数据有表格或文字兜底（不仅依赖颜色区分系列）

## 8. 与工程实现的衔接

| 设计决策 | 工程落点 |
|----------|----------|
| 共享 UI 组件 | `repo/frontends/console/src/components/`（按模块分子目录） |
| 页面 | `repo/frontends/console/src/routes/`（TanStack 文件路由） |
| 全局壳层 | `__root.tsx`：`Layout` + `Header` + `Aside` + `Menu` |
| 主题样式 | `tdesign-react/es/style/index.css` + TDesign 主题变量覆盖 |
| 禁止 | 引入 Ant Design / MUI / 手写平行组件库 |

详细组件与 Token 对照见《TDesign 组件与 Token 2.0》。
