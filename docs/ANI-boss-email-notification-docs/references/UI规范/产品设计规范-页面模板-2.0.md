# 产品设计规范 · 页面模板 2.0

> 本文档定义 ANI Console / BOSS 的页面级模板，帮助设计在新页面开发时快速选型。  
> 所有模板均基于 **TDesign Layout + 标准组件** 实现。

---

## 1. 模板使用规则

设计开始前回答：

1. 用户的主任务是什么？
2. 页面是 **浏览、编辑、决策、监控还是配置**？
3. 是否需要 **列表与详情并列**（检查器）或 **AI 辅助上下文**？
4. 页面主要是 **列表型、表单型还是摘要型**？

**规则：**

- 先选 **最简单** 的模板，再按需加 Secondary Region
- 信息组织服从 **用户任务流**，不是数据库表结构
- 新模板类型须评审通过后再写入本规范，禁止单页发明新壳层

---

## 2. 通用页面骨架

绝大多数页面包含以下区域：

```text
┌──────────────────────────────────────────────┐
│ App Shell（Layout Header + Aside Menu）       │  ← 全局，见 §3
├──────────────────────────────────────────────┤
│ Page Header（标题 + 说明 + 主操作）            │
├──────────────────────────────────────────────┤
│ Content Body（主工作面）                       │
├──────────────────────────────────────────────┤
│ Optional Secondary Region（筛选/检查器/AI）     │
├──────────────────────────────────────────────┤
│ Feedback Region（Alert / Message 挂载点）      │
└──────────────────────────────────────────────┘
```

### 2.1 Page Header

**通常包含：**

- 页面标题（`20px` / semibold）
- 副标题或上下文（`14px` / secondary 色）
- 主次操作按钮组（**唯一** `Button theme="primary"`）
- 可选：`Tabs`、`Breadcrumb` 末级、`Tag` 状态

**TDesign 实现参考：** 自定义 `PageHeader` 组件或 `Space` + `Typography` + `Button`

**规则：**

- 标题必须说明 **当前对象或任务**（「模型管理」✓，「管理」✗）
- 页头不放超过 5 个按钮；低频收进 `Dropdown`

### 2.2 Content Body

承载：摘要卡、表格、图表、表单、详情分组。

### 2.3 Secondary Region

可选，承载：筛选抽屉、详情检查器、预览、活动时间线、AI 面板。

窄屏（<1280px）→ `Drawer` 或下移到主内容下方。

### 2.4 Feedback Region

全局 `Message`；页内 persistent 提示用 `Alert`（配额、权限、只读模式）。

## 3. Console 应用壳层（App Shell）

**适用：** Console、BOSS 全站。

**推荐布局（对齐 `demo-instance-workspace-ui-a` 参考）：**

| 区域 | 规格 | TDesign |
|------|------|---------|
| 顶栏 | 高度 56–64px，品牌色底 | `Layout.Header`，背景 `--td-brand-color` |
| Logo + 产品名 | 左对齐 | 反色文本 `--td-text-color-anti` |
| 租户 / 搜索 / 用户 | 右对齐 | `Dropdown`、`Input` 搜索（P1） |
| 侧栏 | 宽 220–240px，浅色 | `Layout.Aside` + `Menu theme="light"` |
| 内容区 | 剩余宽度，padding 16–24px | `Layout.Content` |

**侧栏 Menu 分组建议（Console）：**

- 概览
- 算力与实例（VM / 容器 / GPU）
- AI 服务（模型 / 推理 / 知识库）
- 存储与网络
- 镜像与 Registry
- 用量与账单
- 设置

BOSS 侧栏分组以 `repo/services/docs/boss-modules/` 为准，壳层结构相同。

## 4. 模板 A：概览页（Dashboard）

### 4.1 适用场景

- Console 首页、BOSS 系统总览
- 健康检查、监控首页

### 4.2 推荐结构

1. Page Header（可选，首页可简化）
2. 核心指标：`Row` + `Col` + `Card` + `Statistic`（4–6 个）
3. 趋势：`echarts` 折线图（如 7 日调用量）
4. 最近活动：`Table` 简表或 `List`

### 4.3 ANI 示例内容（Console）

| 卡片 | 指标 |
|------|------|
| GPU | 总量 / 已用 / 空闲 |
| 推理服务 | 运行中 / 部署中 / 异常 |
| 知识库 | 数量 / 文档数 |
| 用量 | 近 7 日趋势图 |

### 4.4 规则

- 单行摘要卡 **不超过 6 个**
- 图表必须回答明确问题（趋势、占比），不填空白
- 「最近活动」行可点击进详情页

### 4.5 Console 概览页布局间距（定稿）

> 工程实现：`repo/frontends/console/src/components/shell/console-page.css`（`cp-*` 类）

| 层级 | 间距 | 说明 |
|------|------|------|
| 概览页 Content 内边距 | **16px** | `.console-content.console-content--overview { padding: 16px }` |
| 页面栈（`ConsolePage` 直接子元素之间） | **16px** | 概览：`cp-page--overview`（CSS Grid + `gap: 16px`）；列表/详情：`cp-page`（flex + `gap: 16px`） |
| 概览页内栅格 | **16px** | `cp-metrics-grid` / `ov-card-grid` / `ov-quick-grid`；**禁止** TDesign `Row` 承担纵向间距 |
| 列表页 Row gutter（如需要） | **16px（仅水平）** | `gutter={16}` |
| 页头与主内容（列表页） | **16px** | `ConsolePageHeader` + `ConsoleContentCard`，由 `ConsolePage` 栈间距保证 |
| 内容区内边距（非概览） | **20px 24px 32px** | `.console-content`（壳层 Content） |
| Card 外边距 | **0** | `.cp-section-card` / `.cp-stat-card { margin: 0 }`，避免与 `gap` 叠加 |

**结构示意：**

```text
ConsolePage.cp-page--overview（grid, gap 16px）
├── ConsolePageAlert.cp-span-all（可选）
├── cp-metrics-grid.cp-span-all（4 指标卡）
├── ConsoleSectionCard 资源概况          ┐ 宽屏并排，窄屏上下堆叠
├── ConsoleSectionCard 资源使用率        ┘ gap 均为 16px
├── ConsoleSectionCard.cp-span-all 快捷入口
└── ConsoleSectionCard.cp-span-all 告警与待处理
```

**反模式（禁止）：** 概览页多个纵向区块塞进 `cp-dual-grid` / TDesign `Row` 中间层；TDesign Card 默认 margin 或 `Row` 负 margin 会与 `gap` 叠加，造成视觉不等。

## 5. 模板 B：资源列表页

### 5.1 适用场景

- 模型列表、知识库列表、实例列表、镜像列表
- 工单、任务、用户等标准记录集合

### 5.2 推荐结构

1. Page Header + 「创建」主按钮
2. 筛选工具栏：`Form` inline 或 `Input` 搜索 + `Select` 状态
3. `Table` + 行内操作
4. `Pagination`

### 5.3 TDesign 组件

`Button`、`Input`、`Select`、`Table`、`Tag`、`Pagination`、`Empty`、`Loading`

### 5.4 规则

- **必须**设计 loading、empty、error 三态
- 筛选器靠近表格上方
- 操作列固定右侧；删除走 `Dialog` 确认
- 分页与总数信息位置固定，不随数据跳动

### 5.5 ANI 路由示例

| 页面 | 路由 |
|------|------|
| 模型列表 | `/models` |
| 知识库列表 | `/kb` |
| 用量 | `/usage` |
| 镜像 Registry | `/registry` |

## 6. 模板 C：资源详情页

### 6.1 适用场景

- 模型详情、知识库详情、实例详情
- 单对象查看、配置、子资源管理

### 6.2 推荐结构

1. Page Header：对象名 + 状态 `Tag` + 生命周期操作
2. 摘要区：`Descriptions` 或 Card 内 key-value
3. `Tabs`：概览 / 配置 / 日志 / 关联资源
4. Tab 内容：表格、表单或图表

### 6.3 规则

- 核心动作在页头或首屏摘要
- 详情分组按 **用户理解顺序**，不是 API 字段顺序
- 关联资源 Tab 不压过主对象信息

### 6.4 ANI 示例

- 知识库详情 `/kb/:kbId`：文档列表 Tab + 问答 `/kb/:kbId/chat`
- 模型详情 `/models/:modelId`（规划）

## 7. 模板 D：表格 + 检查器（Master-Detail）

### 7.1 适用场景

- 实例工作台、审批队列、审计日志巡检
- 需要 **边看列表边看详情**，减少跳页

### 7.2 推荐结构

```text
┌────────────────┬──────────────────────┐
│ Table / List   │ Detail Panel         │
│ （主列表）      │ （检查器：Tabs/表单）  │
│                │                      │
└────────────────┴──────────────────────┘
```

### 7.3 TDesign 实现

- 左右分栏：`Row` + `Col` 或 flex 布局
- 选中行：背景 `--td-brand-color-light`
- 窄屏：检查器 → `Drawer`

### 7.4 规则

- 行选中态 **必须可感知**
- 检查器强调可扫读、可执行、少跳转
- 行内按钮与行选中 click 不冲突

### 7.5 ANI 参考

`demo-instance-workspace-ui-a`：实例列表 + 详情面板 + 生命周期操作 + 多 Tab（overview / metrics / network / events / audit 等）

## 8. 模板 E：表单页 / 向导页

### 8.1 适用场景

- 新建资源、编辑配置、系统设置
- 多步向导（模型导入、实例创建）

### 8.2 推荐结构

1. Page Header
2. 可选：`Steps`（多步向导）
3. `Form` 主体
4. 底部操作区：取消（default）+ 提交（primary）

### 8.3 规则

- 字段顺序按 **用户决策顺序**
- 必填、选填、help、error 齐全
- 明确 **手动保存** vs **自动保存**（Switch 即时生效须说明）
- 风险操作与保存按钮 **物理分离**

### 8.4 ANI 示例

- 模型导入向导 `/models/import`：`Steps` + `Form`
- API Key 管理 `/settings/api-keys`

---

## 9. 模板 F：活动与审计页

### 9.1 适用场景

- 操作时间线、审计日志、系统事件流
- Core `operations` 时间线展示

### 9.2 推荐结构

1. Page Header
2. 快速筛选：`DateRangePicker` + `Select`
3. 可选：摘要 `Statistic` 或 `Alert`
4. `Table` 或 `Timeline` 主体

### 9.3 规则

- 时间、对象、动作、结果列优先保证扫读
- 失败/风险行：`Tag theme="danger"`，但避免满屏高饱和
- 导出、刷新在页头稳定位置

---

## 10. 可选模式：AI 辅助侧栏

**适用：** 知识库问答、诊断解释、任务引导。

**结构：** 主内容（模板 B/C/D）+ 右侧 `Drawer` 或固定分栏。

**规则：**

- AI 不替代主列表/详情信息架构
- 空态给示例 prompt（`Empty` + 快捷 Chip）
- 处理中不阻断主流程

**ANI 示例：** `/kb/:kbId/chat` 对话区（可演进为侧栏模式）

---

## 11. 模板选型速查

| 用户目标 | 推荐模板 |
|----------|----------|
| 快速了解系统状态 | A 概览页 |
| 管理一类资源集合 | B 资源列表页 |
| 深入单个对象 | C 资源详情页 |
| 高频切换查看多条记录 | D 表格+检查器 |
| 创建或修改配置 | E 表单页 |
| 追溯操作与审计 | F 活动与审计页 |
| 不确定 | 先从 B 或 C 开始 |

---

## 12. 响应式断点建议

| 断点 | 宽度 | 行为 |
|------|------|------|
| 桌面基准 | ≥1280px | 全布局 |
| 紧凑桌面 | 1024–1279px | 侧栏可收窄；检查器变窄 |
| 平板 | 768–1023px | 检查器 → Drawer；表格横滚 |
| 小屏 | <768px | 仅保证可读；BOSS/Console 不以小屏为主设计目标 |
